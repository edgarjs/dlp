# Templates

Ready-to-use templates for DLP output artifacts. Copy and adapt these templates when creating documentation for your project.

## Available Templates

| Template                                                       | Phase        | Purpose                                    |
| -------------------------------------------------------------- | ------------ | ------------------------------------------ |
| [requirements-specification.md](requirements-specification.md) | Requirements | Formal, testable requirements document     |
| [user-stories.md](user-stories.md)                             | Requirements | User story with acceptance criteria        |
| [design-decision.md](design-decision.md)                       | Design       | Record of design exploration and decisions |
| [architecture.md](architecture.md)                             | Design       | System architecture documentation          |
| [data-model.md](data-model.md)                                 | Design       | Entity and relationship definitions        |
| [api-contract.yml](api-contract.yml)                           | Design       | OpenAPI 3.1 specification                  |
| [test-strategy.md](test-strategy.md)                           | Testing      | Test planning and coverage document        |

## Usage

1. Copy the relevant template to your project's documentation directory
2. Replace placeholder text (marked with `[brackets]` or `TODO`)
3. Remove sections that don't apply to your project
4. Follow the guidance in each template's comments
